1. The Namespace (The Room)ArgumentExplanationmetadata.nameThe unique name of the "room."metadata.labels"Stickers" used for organizing or security policies (e.g., env: prod).2. The Deployment (The Manager)ArgumentExplanationreplicasHow many copies (Pods) of the app should run at once.selector.matchLabelsHow the Deployment finds the Pods it is supposed to manage.template.metadata.labelsThe labels given to the Pods (must match the selector above).containers.imageThe Docker image (e.g., mongo:latest or farid/app:v1).ports.containerPortThe internal port the application listens on (e.g., 27017 for Mongo).resources.requestsThe "Minimum" RAM/CPU the pod needs to start.resources.limitsThe "Maximum" RAM/CPU the pod can use before being OOMKilled.env.valueFrom.secretKeyRefLinks an environment variable to a specific key in a Secret.volumeMountsWhere the "Hard Drive" is plugged into inside the container folder.3. The Service (The Switchboard)ArgumentExplanationselectorTells the Service which Pods to send traffic to (based on their labels).type: ClusterIPInternal Only. Only other pods can see this service.type: NodePortExternal. Opens a port on the physical machine (30000-32767).type: LoadBalancerExternal. Requests a public IP from a Cloud provider or Minikube.ports.portThe port the Service listens on.ports.targetPortThe port on the Pod where the traffic is sent.4. The Ingress (The Gatekeeper)ArgumentExplanationannotationsExtra "instructions" for the controller (e.g., ssl-redirect: "true").ingressClassNameTells K8s which controller to use (usually nginx).rules.hostThe domain name (e.g., farid.local).rules.http.paths.pathThe URL sub-folder (e.g., /api or /).rules.http.paths.pathTypeUsually Prefix (matches anything starting with that path).tls.secretNameThe name of the Secret holding your SSL certificates.5. Persistent Volume Claim (The Hard Drive Request)ArgumentExplanationaccessModesReadWriteOnce (1 pod) or ReadWriteMany (Multiple pods/NFS).resources.requests.storageHow much space you want (e.g., 5Gi).storageClassNameThe "Brand" of storage (e.g., standard in Minikube, gp2 in AWS).
